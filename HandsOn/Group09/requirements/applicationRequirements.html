Application Requirements & Mock-ups (Assignment 1)

Scope for this deliverable: select datasets and specify the application idea with UI mock-ups.

Selected datasets

  • Pedestrian Counting System — Monthly counts per hour (City of Melbourne). Hourly pedestrian totals since 2009; updated monthly. The column sensor_id/location_id identifies the sensor.
  • Pedestrian Counting System — Sensor Locations (City of Melbourne). Coordinates, status, notes and directions per sensor; joinable via sensor_id/location_id.

Why these datasets fit the assignment (R1–R6)

  • R1 — Smart city domain: mobility / use of public space. Hourly counts help understand pedestrian activity across city locations.
  • R2 — CSV available: official portal offers direct CSV export.
  • R3 — Open license: Victoria open-data directory indicates CC BY 4.0 (except trademarks/third-party material), suitable for course publishing.
  • R4 — Linkable: every row has a sensor_id; we can join 1-to-1 with the Sensor Locations dataset to add coordinates, status and notes about moves/removals.
  • R5 — Documentation (optional): the resource page includes a Data Dictionary with the column schema.
  • R6 — Multiple sources (optional): we link two official sources from the same portal: Monthly counts per hour + Sensor Locations. (Optional later: enrich with public holidays/events or weather.)

Application concept

Working title: Pedestrian Activity Explorer (Melbourne)

Goal: Explore how foot traffic changes by place and time; compare regular days vs. public holidays; identify peak hours and high-activity locations.

Concrete questions the UI should answer

  • Which Melbourne areas have the highest pedestrian traffic during peak hours?
  • How do counts at a given place differ on a regular day vs. a public holiday?
  • Is there an observable relationship between weather and pedestrian volumes?

Minimal feature list for the prototype

  • Load the two CSVs locally (counts + locations) and join by sensor_id.
  • Filter by date range and time window; aggregate to show averages and peaks.
  • Map of sensors (size/color = intensity) and a side panel with a small time series.
  • Comparator view: regular day vs. holiday for a selected sensor/time window.
